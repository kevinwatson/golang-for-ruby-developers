### Chapter 2 - Ruby on Rails

> 

## Introduction

The Ruby programming language is labeled "A Programmer's Best Friend." It's a dynamic language that provides a concise syntax and has a rich ecosystem of libraries and frameworks like Ruby on Rails that makes it easy to build full applications in a short amount of time.

Ruby on Rails is a framework that runs on the Ruby. It provides the tooling out of the box which makes it easy to stand up a database-backed website or JSON API. It supports multiple databases, generating email, background jobs, WebSockets and JavaScript to name a few features that are needed in modern web apps. Rails powers a number of popular websites, including GitHub, Airbnb, Instacart and Shopify.

## Will it make my job easier?

Before generative LLMs like Claude, Rails generators were the fastest way to generate boilerplate code that you could then customize. With generative LLMs, you can now create both the boilerplate and customized code with a few prompts. Rails provides access to first and third party libraries (aka 'gems' in Ruby), which makes it easy to add and configure the functionality you need in your website or API.

## Will it make my customers happy?

Borrowing from chapter 1:

> Let's define a customer. In the context of this guide, we'll define a customer as anyone who uses your software to perform their job or provide a service to their customers.

Customers are generally unaware of the underlying technologies, languages and frameworks that were used to build the websites and APIs they interact with every day. If they are a customer that can request changes (for example internal users in a company) and you can quickly iterate and deliver new features and bug fixes, they will notice. If they're a customer that interacts with your website and it's responsive, they will notice.
